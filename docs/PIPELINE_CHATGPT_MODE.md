# Transcription Pipeline — ChatGPT-Locked Mode

Last updated: 2026-03-17

This document explains what happens when you click **Run Pipeline**, with emphasis on the legacy ChatGPT-locked mode and the current fallback behavior.

---

## 1) Plain-English Summary (for anyone)

When you click **Run Pipeline**:

1. The app uploads your file (audio or transcript).
2. The server creates a job and tracks progress.
3. If audio, it runs Whisper transcription first.
4. The server sends transcript content to the active LLM path, or to **OpenAI (ChatGPT path)** when locked mode is enabled, to build a DM review packet.
5. The result is stored in the **Approval Queue** as a pending proposal.
6. Nothing is merged into canon until you approve.

Key rule now:

- By default, the pipeline is now flexible and follows the currently selected provider/model.
- If `PIPELINE_CHATGPT_ONLY=true`, the pipeline is forced onto the OpenAI path regardless of the UI provider dropdown.
- UI provider/model settings still exist, but locked mode overrides them for pipeline extraction.

---

## 2) Why this mode exists

Historically, mixed providers and changing prompt paths caused inconsistent extraction quality.

To reduce drift:

- Locked mode sends pipeline calls through one model path (`openai`)
- One dedicated pipeline model is used first (`PIPELINE_OPENAI_MODEL`)
- An OpenAI fallback model can be tried if the primary request fails (`PIPELINE_OPENAI_FALLBACK_MODEL`)
- Proposal records include reviewer provider/model for traceability

---

## 3) Configuration

Environment variables:

- `PIPELINE_CHATGPT_ONLY` (default: `false`)
  - `true` = pipeline uses OpenAI model path only
  - `false` = pipeline uses the generic provider routing

- `PIPELINE_OPENAI_MODEL` (default: `gpt-5.3-chat-latest`)
  - model used by pipeline when locked mode is enabled

- `PIPELINE_OPENAI_FALLBACK_MODEL` (default: `gpt-5-mini`)
  - fallback model used if the primary OpenAI pipeline request fails

- `OPENAI_API_KEY` (required when locked mode is enabled)

- `MAX_UPLOAD_BYTES` (default: `209715200`)
  - upload size limit for audio and transcript files

- `JOB_RETENTION_MS` (default: `1800000`)
  - how long terminal jobs remain queryable through `/api/transcribe/:id`

- `MAX_RETAINED_JOBS` (default: `100`)
  - cap for retained terminal jobs in memory

Operational note:

- If `PIPELINE_CHATGPT_ONLY=true` and `OPENAI_API_KEY` is missing, pipeline extraction fails clearly.

## 4) End-to-end flow (technical)

### Frontend trigger

File: `src/App.jsx`

- `runPipeline()` chooses endpoint by mode:
  - audio → `POST /api/transcribe`
  - transcript → `POST /api/transcribe-text`
- Polls `GET /api/transcribe/:id` until `done` or `error`

### Server endpoints

File: `server.mjs`

- `/api/transcribe` (audio path)
- `/api/transcribe-text` (text path)
- `/api/transcribe/:id` (job status)

### Job lifecycle

- job created in `jobs` Map
- status/stage/progress fields update through pipeline stages
- completed, failed, and cancelled jobs are compacted and eventually evicted from memory

### Audio path internals

`processAudioJob(jobId)`

1. upload audio to remote host over SSH/SCP
2. split into chunks
3. run Whisper per chunk
4. merge transcript + segments
5. run `runLLMStages(job)`
6. queue proposal in approvals
7. link source to selected game session

### Transcript path internals

`processTranscriptJob(job)`

1. read uploaded transcript file
2. run `runLLMStages(job)`
3. queue proposal in approvals
4. link source to selected game session

### LLM extraction stage

`runLLMStages(job)`

- Builds canon context
- Calls `llmGeneratePipeline(prompt)`
- Parses strict JSON extraction fields
- Normalizes NPC/quest names vs canon
- Creates proposal object and writes to approvals queue

### Pipeline LLM router

`llmGeneratePipeline(prompt)`

- If `PIPELINE_CHATGPT_ONLY=true`:
  - calls `openaiGenerate(prompt, PIPELINE_OPENAI_MODEL)` first
  - may retry with `PIPELINE_OPENAI_FALLBACK_MODEL`
- Else:
  - calls generic `llmGenerate(prompt)`

### Diarization labels

`diarizeSegmentsGuess(segments)` also uses `llmGeneratePipeline(...)` now,
so speaker-label inference follows the same ChatGPT-locked path.

## 5) Data artifacts (where output is stored)

Campaign directory: `data/campaigns/<campaignId>/`

- `approvals.json` — pending proposals generated by pipeline
- `game-sessions.json` — source linkage to sessions
- `imports/` — pre-AI artifacts, checkpoints, and diarization outputs
- `sessions/` — raw approved proposal snapshots
- `journal.json` / `story-journal.json` — updated only after approval

Proposal metadata now includes:

- `reviewerProvider`
- `reviewerModel`

for better auditability.

---

## 6) What health/config APIs show

- `GET /api/health` returns:
  - `pipelineChatgptOnly`
  - `pipelineOpenaiModel`
  - `pipelineOpenaiFallbackModel`
  - current diarization mode and key presence indicators

- `GET /api/llm/config` returns those same pipeline flags

- `PUT /api/llm/config` now returns a note when pipeline is locked,
  clarifying that dropdown changes do not control pipeline extraction.

- `GET /api/pipeline/key` reports whether an OpenAI key is present for locked mode.

- `PUT /api/pipeline/key` updates the persisted OpenAI key used by locked mode.

---

## 7) How to step through in code

1. Frontend trigger: `src/App.jsx` → `runPipeline()`
2. Endpoint handler: `server.mjs` → `/api/transcribe` or `/api/transcribe-text`
3. Worker path: `processAudioJob` / `processTranscriptJob`
4. Extraction core: `runLLMStages`
5. LLM router: `llmGeneratePipeline`
6. Provider call: `openaiGenerate`
7. Proposal write: `queueApproval(...)`
8. UI poll output: `/api/transcribe/:id`

Tip for debugging one run:

- start from jobId returned by POST endpoint
- watch `/api/transcribe/:id` stage/progress
- inspect latest entry in `approvals.json`
- inspect `imports/` if you need pre-AI transcripts or intermediate checkpoint payloads
- remember that old jobs may disappear after the retention window expires

---

## 8) Expected behavior after this change

- Pipeline uses OpenAI path consistently when locked mode is enabled
- Pipeline can fall back to the configured secondary OpenAI model in locked mode
- Proposal reviewer metadata is no longer missing on new runs
- Team members can verify model path from API health/config

---

## 9) Known limitations

- Single-pass structured extraction can still produce weak quality if transcript quality is poor.
- This mode improves consistency of model path, not guaranteed content quality.

If quality tuning is needed, change prompt/schema deliberately and re-test with fixed fixture transcripts.
